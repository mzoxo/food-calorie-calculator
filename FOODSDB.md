# 食材資料庫瀏覽頁 — 規格說明

> 路由：`/foods`
> 元件：`src/views/FoodsView.vue`

---

## 一、入口

- AppHeader 右側加入「食材庫」按鈕（或 AppMenu 內）
- 點擊進入 `/foods` 頁面

---

## 三、功能規格

### 3.1 搜尋
- 即時過濾，debounce 300ms
- 匹配欄位：名稱、別名、品牌
- 有搜尋詞時顯示清除按鈕

### 3.2 分類篩選
- 從食材資料動態撈出所有不重複分類
- 下拉選單，預設「全部」
- 可與搜尋同時作用（AND 邏輯）

### 3.3 排序
- 可排序欄位：熱量、碳水、蛋白質、脂肪（皆為每 100g）
- 點擊欄位標頭切換：無 → 升冪 ↑ → 降冪 ↓ → 無
- 同時只能有一個欄位排序
- 預設：依名稱字母順序

### 3.4 加入計算頁
- 每列右側 `＋` 按鈕
- 點擊開啟 AddFoodModal（與主計算頁相同邏輯）
- 加入後顯示 Toast 提示

### 3.5 比較模式
切換比較模式後：
- 每列左側出現 checkbox
- 最多選取 **5 項**食材
- 底部浮出列顯示「已選 N 項」+ 開始比較按鈕

## 四、比較 Modal 規格

### 4.1 基準設定
使用者選擇比較基準：

| 基準 | 說明 |
|------|------|
| 固定克數 | 各食材皆以 X 克換算，預設 100g |
| 固定熱量 | 各食材達到 X kcal 需要幾克 |
| 固定蛋白質 | 各食材達到 X g 蛋白質需要幾克 |
| 固定碳水 | 各食材達到 X g 碳水需要幾克 |
| 固定脂肪 | 各食材達到 X g 脂肪需要幾克 |

### 4.2 結果顯示
橫向表格，每欄一個食材，列出：
- 需要克數（依基準換算）
- 熱量
- 碳水
- 蛋白質
- 脂肪
- 膳食纖維

### 4.3 加入計算頁
- 每個食材欄位下方有「加入」按鈕
- 點擊以換算後的克數加入 AddFoodModal
- 可選群組後確認加入

## 六、資料來源

- 直接使用 `store.foods`（已載入的食材庫）
- 不需額外 API 呼叫
- 若 `store.foods` 為空，顯示「食材資料尚未載入，請返回首頁」
